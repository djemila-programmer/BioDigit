🧪 Simulation Sans Biodigesteur Réel

Comment simuler les conditions d'un biodigesteur

🌡️ Température (DHT22)

Simuler avec :
• Verre d'eau chaude (35-40°C) → zone optimale
• Verre d'eau froide (25°C) → zone basse
• Glaçon dans l'eau → chute de température
• Frotter la sonde → augmentation légère

Plage biodigesteur : 25°C - 40°C
Optimal : 30°C - 38°C (mésophile)

📊 Pression (BMP280)

Simuler avec :
• Pression ambiante normale (~1 bar)
• Souffler doucement sur le capteur
• Monter en altitude (ascenseur, étage)
• Le capteur lit la pression atmosphérique

Plage biodigesteur : 0.8 - 1.5 bar
Normal : ~1.0 bar

💨 Méthane (MQ-4)

Simuler avec :
• Allumette éteinte près du capteur (gaz)
• Briquet sans flamme (butane)
• Vinaigre près du capteur (réaction)
• Air frais → valeur basse

⚠️ Chauffer le MQ-4 pendant 24h avant première utilisation !
Plage biodigesteur : 150 - 500 ppm
Normal : ~300 ppm

📏 Niveau (HC-SR04)

Simuler avec :
• Main devant le capteur → niveau haut
• Éloigner la main → niveau bas
• Livre/carton à distance fixe → niveau stable
• Rapprocher/éloigner → variation

Plage biodigesteur : 20% - 90%
Formule : niveau = (100 - distance) / 100 × 100

🚀 Étapes de Mise en Route (Simulation)

  1. Acheter le matériel (~21 000 FCFA) — voir liste ci-dessus
  2. Créer le projet Supabase sur supabase.com (gratuit)
  3. Créer les tables : sensor_readings, esp32_status, profiles (SQL dans le README)
  4. Activer Realtime sur sensor_readings et esp32_status
  5. Configurer les politiques RLS pour sécuriser les données
  6. Récupérer SUPABASE_URL et SUPABASE_ANON_KEY
  7. Brancher les capteurs selon le schéma ci-dessous
  8. Installer Arduino IDE + bibliothèques (DHT sensor library, Adafruit BMP280, ArduinoJson)
  9. Modifier le code ESP8266 : WiFi SSID/password, Supabase URL/Key, User ID
  10. Téléverser le code sur l'ESP8266 via USB
  11. Ouvrir le moniteur série (115200 bauds) → vérifier les lectures
  12. Simuler : plonger DHT22 dans eau chaude, approcher allumette du MQ-4, etc.
  13. Lancer l'application Flutter → se connecter → voir les données en temps réel
  14. Tester les alertes : dépasser les seuils pour voir les notifications

📌 Schéma de Câblage Complet

ESP8266 DevKit V1
3V3 D0 (GPIO 16) D3 (GPIO 0) D4 (GPIO 2) D7 (GPIO 13)
GND D8 (GPIO 15) D2 (GPIO 4) → DHT22 A0 VIN (5V)
D1 (GPIO 5) → BMP SCL RX (GPIO 3) TX (GPIO 1) GND 5V (VIN)
D5 (GPIO 14) → HC Trig D6 (GPIO 12) → HC Echo A0 → MQ-4 SD3 (GPIO 10) SD2 (GPIO 9)

Vert = Utilisé   Rouge = Alimentation   Gris = GND   Foncé = Libre

🔄 Flux de Données - Du Capteur à l'Application

📊 Format des Données Envoyées à Supabase

POST https://VOTRE-PROJET.supabase.co/rest/v1/sensor_readings
Headers:
  Authorization: Bearer VOTRE_CLE_ANON
  Content-Type: application/json

Body:
{
  "user_id":        "a1b2c3d4-e5f6-...",
  "temperature":    36.5,
  "pressure":       1.05,
  "methane":        320,
  "slurry_level":   72.5,
  "temperature_trend": "stable",
  "pressure_trend":   "rising",
  "methane_trend":    "stable",
  "slurry_trend":     "falling"
}

🗄️ Tables Supabase Requises

📋 sensor_readings
id (BIGSERIAL PK) user_id (UUID FK) temperature (FLOAT) pressure (FLOAT) methane (FLOAT) slurry_level (FLOAT) temperature_trend (TEXT) pressure_trend (TEXT) methane_trend (TEXT) slurry_trend (TEXT) created_at (TIMESTAMPTZ)

⚡ Realtime activé → pousse les inserts à l'app

📋 esp32_status
id (BIGSERIAL PK) user_id (UUID FK) connected (BOOLEAN) wifi_signal (INT) firmware_version (TEXT) battery_level (INT) ip_address (TEXT) uptime (TEXT) cpu_temp (FLOAT) last_sync (TIMESTAMPTZ)
📋 profiles
id (UUID PK = auth.users.id) full_name (TEXT) email (TEXT) phone (TEXT) farm_name (TEXT) role (TEXT) → 'user' | 'admin' location (TEXT)
