📏 HC-SR04 - Niveau Ultrason

Broche HC-SR04→ ESP32
VCC5V (VIN) ⚠️ 5V requis!
GNDGND
TrigGPIO 5
EchoGPIO 18 ⚠️ Diviseur!
Echo sort 5V, GPIO18 accepte 3.3V max !
Diviseur : Echo → [1kΩ] → GPIO18 → [2kΩ] → GND

⚡ Diviseur de Tension (HC-SR04 Echo → ESP32)

  HC-SR04 Echo (5V)
       │
      [1kΩ]
       │
       ├──────────→ GPIO 18 (ESP32)  ← 3.3V ✓
       │
      [2kΩ]
       │
      GND

  Calcul : 5V × 2kΩ/(1kΩ+2kΩ) = 3.3V ✓

  Authorization: Bearer VOTRE_CLE_ANON
  Content-Type: application/json

Body:

  "pressure":       1.05,
  "methane":        320,

  "pressure_trend":   "rising",
  "methane_trend":    "stable",

🗄️ Tables Supabase Requises

📋 sensor_readings
id (BIGSERIAL PK) user_id (UUID FK) temperature (FLOAT) pressure (FLOAT) methane (FLOAT) slurry_level (FLOAT) temperature_trend (TEXT) pressure_trend (TEXT) methane_trend (TEXT) slurry_trend (TEXT) created_at (TIMESTAMPTZ)

⚡ Realtime activé → pousse les inserts à l'app

📋 esp32_status
id (BIGSERIAL PK) user_id (UUID FK) connected (BOOLEAN) wifi_signal (INT) firmware_version (TEXT) battery_level (INT) ip_address (TEXT) uptime (TEXT) cpu_temp (FLOAT) last_sync (TIMESTAMPTZ)
📋 profiles
id (UUID PK = auth.users.id) full_name (TEXT) email (TEXT) phone (TEXT) farm_name (TEXT) role (TEXT) → 'user' | 'admin' location (TEXT)

🚀 Étapes de Mise en Route

  1. Créer le projet Supabase sur supabase.com
  2. Créer les tables : sensor_readings, esp32_status, profiles (SQL dans le README)
  3. Activer Realtime sur sensor_readings et esp32_status
  4. Configurer les politiques RLS pour sécuriser les données
  5. Récupérer SUPABASE_URL et SUPABASE_ANON_KEY
  6. Créer un compte utilisateur dans l'application Flutter (inscription)
  7. Copier l'UUID de l'utilisateur (Supabase Dashboard → Authentication → Users)
  8. Brancher les capteurs selon le schéma ci-dessus
  9. Installer Arduino IDE + bibliothèques (OneWire, DallasTemperature, Adafruit BMP280, ArduinoJson)
  10. Modifier le code ESP32 : WiFi SSID/password, Supabase URL/Key, User ID
  11. Téléverser le code sur l'ESP32
  12. Ouvrir le moniteur série (115200 bauds) → vérifier les lectures
  13. Lancer l'application Flutter → se connecter → voir les données en temps réel
